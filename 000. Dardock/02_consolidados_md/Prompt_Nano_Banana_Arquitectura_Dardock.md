# Prompt Nano Banana — Arquitectura Digital Dardock

```text
Quiero que diseñes una arquitectura digital clara, simple y escalable para una agencia boutique llamada Dardock.

CONTEXTO
Dardock opera como Growth Partner + Performance Ads agency.
Trabaja con:
- Meta Ads
- Google Ads
- dashboards de auditoría
- landing pages
- funnels comerciales
- CRM y automatizaciones

Actualmente usamos o evaluamos estas plataformas:
- GitHub
- Vercel
- GoHighLevel (GHL)
- Squarespace
- Hostinger
- Google Workspace

PROBLEMA
Hoy hay demasiadas plataformas cruzadas y necesito una arquitectura maestra que responda:
- qué vive en cada sistema
- qué no debe vivir ahí
- cómo se conectan entre sí
- cuál es el flujo correcto de trabajo
- cómo evitar caos técnico y operativo

OBJETIVO
Quiero una arquitectura digital ejecutiva, limpia y realista para una agencia pequeña de 2 personas.
No quiero una arquitectura enterprise falsa.
Quiero una estructura pragmática, con pocos sistemas, roles claros y orden.

ENTREGABLE ESPERADO
Diseña la arquitectura en 6 capas:

1. Código y memoria del sistema
2. Publicación y hosting
3. Funnels y CRM
4. Dominio y DNS
5. Correo y colaboración
6. Operación diaria

PARA CADA CAPA DEFINE:
- plataforma recomendada
- función principal
- qué debe vivir ahí
- qué no debe vivir ahí
- cómo se conecta con las otras capas
- riesgos si se usa mal

PLATAFORMAS A ASIGNAR
- GitHub
- Vercel
- GHL
- Squarespace
- Hostinger
- Google Workspace

CASOS REALES A CUBRIR

1. Dashboard de auditoría para clientes
2. Landing de Dardock
3. Funnel de captación y reserva de diagnóstico
4. Gestión de leads
5. Documentación interna y playbooks
6. Publicación por subdominio como `central.dardock.com`

DECISIONES QUE QUIERO QUE TOMES

- Qué sistema debe ser la fuente de verdad
- Qué sistema publica
- Qué sistema automatiza
- Qué sistema administra DNS
- Qué sistema maneja correo
- Cuál es el flujo correcto cuando un usuario entra a una landing, agenda una auditoría y luego recibe seguimiento comercial

CRITERIOS
- mínimo caos
- mínimo mantenimiento
- alta claridad operativa
- stack razonable para una agencia pequeña
- fácil de mantener entre varias computadoras
- sin sobreingeniería

SALIDA
Quiero:

1. un mapa arquitectónico resumido
2. una explicación por plataforma
3. una propuesta de flujo de trabajo real
4. un listado de errores comunes a evitar
5. una recomendación final de stack ideal Dardock 2026

TONO
Directo, ejecutivo, claro, técnico pero entendible.
Nada de humo.
Nada de arquitectura enterprise ficticia.
Quiero criterio real de operador.
```
