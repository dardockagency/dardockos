# Mapa Input → Procesamiento → Output — Dardock

## Objetivo

Ordenar cómo entra información al sistema Dardock, cómo se procesa y qué tipo de output debe salir.

Esto evita mezclar:

- contexto
- criterio
- entregables

---

## Resumen ejecutivo

El sistema Dardock funciona mejor cuando separa tres capas:

1. input
2. procesamiento
3. output

Si esas capas se mezclan, aparecen:

- diagnósticos confusos
- propuestas débiles
- reportes poco útiles
- tareas sin foco

---

## 1. Input

### Qué entra al sistema

- contexto del cliente
- métricas
- problemas reportados
- historial de campañas
- estado del tracking
- oferta y funnel
- referencias estratégicas

### Ejemplos de input

- “el CPA subió 40%”
- “el cliente quiere escalar”
- “la landing no convierte”
- “Meta y Shopify no muestran lo mismo”
- “tenemos nuevas piezas creativas”

### Regla

El input no decide nada por sí solo.

Solo alimenta el análisis.

---

## 2. Procesamiento

### Qué pasa en esta capa

El sistema Dardock filtra el input usando:

- identidad central
- operating system
- framework diagnóstico por capas
- matriz síntoma → causa probable
- lógica comercial
- criterio estratégico

### Preguntas que resuelve esta capa

- ¿Cuál es el problema real?
- ¿En qué capa está la fuga?
- ¿Qué sí es prioridad?
- ¿Qué no corresponde hacer todavía?
- ¿Qué tipo de output debe generarse?

### Regla

Aquí se toma la decisión.

No en el input.

---

## 3. Output

### Qué sale del sistema

Según el caso, el output puede ser:

- auditoría
- propuesta
- backlog de optimización
- reporte mensual
- recap
- brief creativo
- ajuste de campaña
- recomendación estratégica

### Regla

El output debe corresponder al tipo de problema procesado.

No todo input termina en propuesta.

No todo input termina en tarea.

No todo input termina en reunión.

---

## Ejemplos completos

### Caso 1

**Input:**  
CPA alto y CVR bajo.

**Procesamiento:**  
revisión por capas muestra buena atracción pero mala conversión post-clic.

**Output:**  
diagnóstico de funnel + recomendación sobre landing y continuidad mensaje-destino.

---

### Caso 2

**Input:**  
cliente quiere aumentar presupuesto.

**Procesamiento:**  
la señal de tracking aún no está validada y el funnel tiene fricción.

**Output:**  
no se recomienda escalar todavía; se propone corrección técnica y optimización previa.

---

### Caso 3

**Input:**  
nuevas piezas creativas listas.

**Procesamiento:**  
se revisa si responden a una hipótesis y a qué etapa del funnel apuntan.

**Output:**  
plan de test creativo, no simple publicación de piezas.

---

## Traducción al sistema actual

### Inputs frecuentes

- métricas
- mensajes del cliente
- resultados de campañas
- hallazgos de tracking
- nuevas ofertas

### Procesadores activos

- `01_uso_diario/DARDOCK_OPERATING_SYSTEM.md`
- `02_consolidados_md/DARDOCK_MASTER_PLAYBOOK.md`

### Outputs principales

- `04_comercial_propuestas/`
- `01_uso_diario/dardock_landing.html`
- reportes y recaps derivados de operación

---

## Regla final

Una agencia desordenada reacciona al input.

Una agencia sólida procesa primero y produce el output correcto después.
