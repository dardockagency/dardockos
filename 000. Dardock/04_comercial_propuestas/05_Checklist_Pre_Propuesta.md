# Checklist Pre-Propuesta Dardock

## No enviar propuesta sin validar esto

- [ ] Tipo de negocio definido
- [ ] Oferta principal identificada
- [ ] Ticket promedio o rango conocido
- [ ] Margen aproximado o al menos viabilidad económica entendida
- [ ] Canal principal de conversión identificado
- [ ] Estado del tracking entendido
- [ ] Principal quiebre del funnel identificado
- [ ] Capacidad operativa del cliente validada
- [ ] Objetivo de negocio del próximo trimestre definido
- [ ] Presupuesto de medios actual o estimado conocido

---

## Elegir propuesta correcta

- [ ] Auditoría
- [ ] Setup Tracking + Base Performance
- [ ] Retainer Performance Core
- [ ] Retainer Growth System

---

## Señales de alerta

- [ ] Cliente quiere escalar sin datos
- [ ] Cliente no tiene oferta clara
- [ ] Cliente no tiene capacidad operativa para absorber demanda
- [ ] Cliente espera resultados sin implementar cambios
- [ ] Cliente pide demasiados servicios fuera del foco performance

Si aparecen 2 o más, ajustar el alcance o no avanzar.
